Process RSVP Tagging (Record Responses)

Filtering CSV File

To help keep track of the records you have already completed it is recommended to use the filtering function in Excel.

Access Filters in Excel: Sort & Filter > Filter > Click arrow in column header

  1. First Report - No need to filter

  2. All Subsequent Reports - Filter the first column "created at" after the date you left off updating (filter picklist, use date filters > after...)

 

Auto-Tagged

  • If "sf update status" column contains a date, the entry was auto-tagged (response was automatically recorded) and you can disregard these entries. However, you will still need to check for potential contact record updates.
  • If "sf update status" column is "NOT updated!", the entry was not auto-tagged and you will need to update manually depending on the scenario (see below)

 

RSVP = YES (contact in Salesforce)

  1. Lookup the contact in Salesforce
  2. Are they a competitor (see Competitor)?
    1. Is competitor - tag response as "Rejected" (for appropriate child campaign)
    2. Not competitor - tag response as "RSVP - Yes" (for appropriate child campaign) and any additional fields like Speaker/Panelist, Comments, etc.

 

RSVP = YES (contact not in Salesforce)

  1. Determine if new contact is a competitor (confirm with the officer)
    1. Is competitor
      1. Create contact record in Salesforce
      2. Add contact to campaign (for appropriate child campaign)
      3. Tag response as "Rejected"
    2. Not competitor
      1. Create contact record in Salesforce
      2. Add contact to campaign (for appropriate child campaign)
      3. Tag response as "RSVP - Yes" and any additional fields like Speaker/Panelist, Comments, etc.

 

RSVP = NO (contact in Salesforce)

  1. Lookup the contact in Salesforce
  2. Tag response as "RSVP - No" (for appropriate child campaign)
