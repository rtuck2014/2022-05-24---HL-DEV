Submitting Conflicts Checks (11/19)

Initial Submission

Interested parties include the client, subject and so-called additional interested parties (that is, any other party that may be relevant to the opportunity or engagement, such as likely transaction counterparties, as well as significant shareholders, lenders and other affiliates of the transaction participants).

  1. Lookup the opportunity or engagement in Salesforce (see Searching)
  2. <Conflicts Check> (top of the page)
  3. Click in the Interested Parties section to add additional interested parties (beyond the Client/Subject company(s)) that need to be included in the conflicts check. It is imperative to follow established guidelines (See Guidelines) when submitting additional interested parties
  4. Answer any questions that appear below the Interested Parties list
  5. When completed, click <Submit>

Interested Party Import Process

If you have a large number of interested parties to add, save time by using the import function.

Complete the Import Template (Step 1)

  1. Download the Interested Party Import Template located on the home page of the Conflicts Check Application via HL Intranet
  2. Enter Interested Party information into the template
  3. Remove Row 2 from the completed template (this row provides notes and should not be imported)
  4. <Save> the completed template

Upload the Import Template (Step 2)

  1. In the Interested Parties area of the submission page, click
  2. <Browse...>
  3. Select the completed template file to upload
  4. <Upload File>
  5. Verify the information then click <Continue>
  6. <Import>
  7. <Return to Request Info> and continue with the submission process

You will receive a Validation Error if the value entered into the template doesn't match valid options. To fix the error, click the "Edit" link next to the validation error. Update the appropriate field in question then click the "Update" link. Once all validation errors have been fixed, click <Update> then <Import>

Withdraw

Assistants and deal team members can withdraw a request any time before a final resolution status is entered.

  1. Go to the Conflicts Check Application accessible on the home page of the HL Intranet
  2. If needed, use the filter options to lookup the conflict check to withdraw
  3. Click the "Open" link
  4. <Withdraw>

Resubmit

Some opportunities may require multiple conflicts checks to be submitted, usually at different deal stages (e.g. in-court restructuring, transaction opinions, litigation, etc.) Please refer to the Conflicts Check Policy located on the HL Intranet (Corporate > Legal > Policies and Procedures).

When you submit a conflicts check for an opportunity or engagement with previous completed conflicts checks, you will typically use one of two resubmission options:

  1. Continue Without Copying - does not copy any interested parties from the previous conflicts check
  2. Copy - copies the interested parties from the previous conflicts check. This option is useful if you wish to add additional parties to a previously submitted conflicts check

Returned Requests

If the Conflicts Check Administrator finds an issue with the conflicts check request, they may send it back to the requester via email notification.

Checking the Status

  1. Lookup the opportunity or engagement in Salesforce
  2. Scroll down to the Conflicts Check section (see example below)

View All of Your Conflicts Check Requests

You can view all of your conflicts check requests from the Conflicts Check Application.

  • Financial Staff - can view requests if they are part of the deal team
  • Admin Staff - can view requests they submitted as well as any request that their officer (whom they are a delegate) is on the deal team
  1. Go to the Conflicts Check Application accessible on the home page of the HL Intranet
  2. If needed, use the filter options to narrow down the list of requests
